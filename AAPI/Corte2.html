AAPI: Software en automatización de análisis y procesos industriales para la reducción de pérdidas en producciones alimentarias.

1. Descripción del Problema

Actualmente, la eficiencia en el contexto industrial se ha convertido en un factor determinante para la competitividad a nivel global. Se ha evidenciado un déficit en los procesos industriales: fallas operativas en el control de variables como temperatura, tiempos de proceso o manipulación inadecuada generan pérdidas del producto y disminución de la capacidad efectiva de producción.

En el sector alimentario, la FAO señala que "el 46% de los desperdicios se da en las etapas de procesamiento, distribución y consumo de alimentos" (FAO, 2021), evidenciando la necesidad urgente de mejorar los sistemas de control y análisis dentro de las plantas productivas.

Ante este escenario, el proyecto AAPI se plantea como una iniciativa orientada a fortalecer los mecanismos de supervisión y evaluación en plantas del sector alimentario, reuniendo y organizando la información operativa para reducir pérdidas.

Pregunta de Investigación

¿De qué manera la implementación de un software automatizado de análisis de procesos puede contribuir a la reducción de pérdidas productivas en las plantas del sector alimentario?

2. Objetivos

Objetivo General

Crear una herramienta de automatización y análisis de procesos industriales para las plantas del sector alimentario que identifiquen y reduzcan pérdidas productivas.

Objetivos Específicos

01

Analizar las variables operativas del proceso industrial alimentario (temperatura, tiempos de operación, desempeño de maquinaria) para determinar su incidencia en la eficiencia productiva.

02

Definir los requerimientos funcionales y técnicos del sistema AAPI mediante criterios de monitoreo, registro y visualización de información operativa en plantas del sector alimentario.

03

Evaluar el desempeño del sistema AAPI mediante pruebas funcionales e indicadores operativos que permitan verificar mejoras en el control y seguimiento de los procesos productivos.

3. Justificación

Académica

Integra los saberes del primer semestre de forma práctica. Se aplicó HTML y CSS para maquetar el sitio completo, simulando el ciclo de un producto tecnológico desde la identificación del problema hasta el prototipo funcional.

Tecnológica

Demuestra que con el stack web básico se construye una herramienta de soporte a la decisión industrial. La arquitectura está lista para consumir una API o comunicarse con un PLC en fases futuras alineadas con Industria 4.0.

Social

Se alinea con la meta 12.3 de los ODS sobre reducción de desperdicio (Naciones Unidas, 2017). Ofrece monitoreo accesible a PYMES alimentarias colombianas que no pueden costear un SCADA propietario.

Económica

Reducir pérdidas de materia prima mejora la rentabilidad y disminuye costos de disposición de residuos. Al usar tecnologías web estándar, no requiere licencias costosas y funciona en equipos existentes.

4. Delimitación

Temática

El alcance se centra en el desarrollo del front-end: un dashboard interactivo donde el usuario ingresa variables (temperatura, humedad, velocidad, tiempo de ciclo) y un motor de reglas en JavaScript genera un diagnóstico de riesgo y estimación de eficiencia. No se incluyen conexiones con sensores reales, bases de datos ni autenticación en esta fase.

Geográfica

El desarrollo se realiza en Bogotá D.C. utilizando computadores personales y los laboratorios de la Universidad San Buenaventura. La solución está pensada para plantas de la Sabana de Bogotá, aunque en este corte no se realizaron visitas técnicas ni pruebas en entornos reales.

Temporal

El proyecto se ejecuta durante el primer semestre de 2026 (febrero–mayo), desarrollando las fases de análisis, diseño y maquetación web. Las siguientes fases —integración con hardware y manejo de datos— están planificadas para los semestres quinto y sexto del programa integrador.

Poblacional

El sistema está dirigido al personal operativo de PYMES del sector alimentario: supervisores, jefes de producción y técnicos de mantenimiento. La interfaz usa colores tipo semáforo, iconos y ayudas visuales para que sea intuitiva sin requerir formación técnica en programación.

5. Limitaciones

Tiempo disponible

El desarrollo se realiza dentro de un semestre académico, lo que limita la implementación de todas las funcionalidades previstas y el margen para pruebas extensas. El prototipo prioriza objetivos básicos y funcionales, dejando componentes avanzados para etapas posteriores.

Acceso a información real

La dificultad para obtener datos concretos de procesos industriales alimentarios obliga a construir el sistema a partir de supuestos y escenarios simulados, generando posibles diferencias entre los resultados del prototipo y la realidad de una planta.

Recursos tecnológicos

La ausencia de equipos especializados o software avanzado limita las posibilidades técnicas del sistema, afectando tanto el diseño como el rendimiento del prototipo. El desarrollo se enfoca en herramientas accesibles como tecnologías web básicas para garantizar la viabilidad del proyecto.

Disponibilidad de usuarios

La baja participación de usuarios para realizar pruebas dificulta validar la usabilidad y funcionalidad del sistema de manera completa, limitando la retroalimentación sobre la experiencia en contextos reales.

Acceso a plantas industriales

Al no trabajar en entornos reales, toda la validación se realiza mediante simulaciones controladas. Los datos no reflejan completamente la dinámica de una planta de producción real. La validación en campo se plantea como objetivo para fases futuras del proyecto integrador.
